# Release Notes v0.6.0 - Triluga UX Mobile e Anúncios

Data: `2026-04-13`

## Resumo

Esta versão consolida os ajustes de experiência mobile, busca, detalhe do veículo e criação de anúncios. O mapa sai da listagem e passa a aparecer somente no detalhe do anúncio, enquanto o editor de anúncio ganha upload de imagens visível desde a abertura.

## Entregas principais

- navegação mobile por arraste lateral entre início, busca, anúncio e chat
- CTA fixo do detalhe do veículo acima da navegação inferior
- mapa de localização restrito à página de detalhe do veículo
- remoção da feature de abrir mapa na busca e no modal de filtros
- botão para salvar alerta de busca no card de alertas
- correção de responsividade nos cards de veículo e badges mobile
- melhoria do editor de anúncio com upload de fotos sempre visível
- botão de voltar no editor de anúncio no padrão da tela de login
- refinamento do chat vazio e ícones de navegação
- remoção do card "Retire" do fluxo da home
- correção da consulta de CEP no backend usando `node:https`
- remoção da dependência `leaflet` não utilizada no frontend

## Impacto esperado

- fluxo de criação de anúncio mais claro no mobile
- menor confusão entre filtros, busca e mapa
- navegação mobile mais fluida sem prejudicar inputs, botões ou carrosséis
- detalhe do veículo com localização mais contextual para o locatário
- consulta de CEP compatível com o runtime Node atual do backend

## Validação

- `npm --prefix frontend run build`
- `npm --prefix backend run build`
- `npm --prefix backend test -- --runInBand`
