# Release Notes v2.2.0 - Triluga Web Marketplace

Data: `2026-04-30`

## Resumo

A versão 2.2 consolida a camada web do Triluga com novas telas desktop, navegação padrão, comparação completa, detalhes de veículo orientados a classificado e melhorias no fluxo de publicação de anúncios.

## Entregas principais

- telas web de favoritos, perfil público, detalhes de veículo e comparação aplicadas sem alterar o layout mobile
- header web com chat, notificações, perfil público, configurações e logout funcional
- footer web padronizado com links do cabeçalho e textos em português
- página de comparação web com tabela funcional usando veículos selecionados em favoritos, busca ou detalhe
- detalhe de veículo web com galeria, preview de imagens, Google Maps, descrição do anunciante, seguro, condição mecânica e status Detran
- perfil público web com cards, anúncios, reputação e tempo médio de resposta calculado a partir do chat
- busca web com filtro de cidade, categorias sem seleção padrão, slider de preço mínimo/máximo e filtros técnicos
- home web com imagens locais de categoria, estatísticas vindas do backend e cards de veículo reaproveitados
- publicação de anúncios exigindo mínimo de 3 fotos, título automático e ações de publicar/despublicar/excluir anúncio
- backend com contagem de visualizações, estatísticas da home, novos filtros de veículos e ordenação por relevância/preço/data

## Impacto esperado

- experiência web mais próxima das telas aprovadas pelo design
- menos duplicação visual entre páginas por uso de header, footer e cards compartilhados
- comparação de veículos mais útil para decisão antes de iniciar contato
- dados públicos de perfil mais confiáveis, com tempo de resposta baseado em mensagens reais
- anúncios mais completos e seguros por exigência mínima de fotos e campos de procedência

## Validação

- `npm --prefix backend run build`
- `npm --prefix frontend run build`

## Observações

- O frontend compila com avisos de budget CSS em componentes já acima do limite configurado: `app`, `home`, `favorites`, `profile`, `user-profile`, `vehicle-detail` e `compare`.
- Esta versão inclui migration para `Vehicle.viewsCount`; aplicar `npm run backend:prepare` ou `npm --prefix backend run prisma:deploy` no ambiente de destino antes de subir o backend.
